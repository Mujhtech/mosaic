# Mosaic Vision

## Mission

Mosaic helps app developers create, ship, and improve native monetization experiences without rebuilding paywalls manually or surrendering control of their UI.

## Product Vision

Mosaic will become the open monetization platform for mobile applications.

It will combine:

- native paywalls
- remote configuration
- placements
- targeting
- publishing
- experimentation
- analytics
- billing-provider integrations
- extensibility
- self-hosting

into one developer-focused system.

## Initial Product Position

Mosaic begins as a native paywall platform, not as a complete subscription backend.

The initial product promise is:

> Create, publish, and update production-quality native paywalls across Flutter, SwiftUI, and Jetpack Compose without releasing a new app version.

## Target Users

Mosaic initially targets:

- independent mobile developers
- early-stage startups
- small mobile teams
- agencies
- teams without dedicated growth engineers
- teams already using RevenueCat
- developers currently hardcoding paywalls

## Core Differentiation

Mosaic combines:

- visual editing
- native rendering
- custom native components
- provider independence
- open-source infrastructure
- self-hosting
- cross-platform consistency
- strong local-development tooling

The main differentiator is:

> Visual editing without losing native UI control.

## Long-Term Direction

Mosaic may later expand into:

- subscription infrastructure
- entitlements
- receipt validation
- customer lifecycle automation
- advanced revenue analytics
- AI-assisted optimization
- onboarding experiments
- feature gating
- template and component ecosystems

Expansion should follow validated user demand rather than being assumed from the beginning.
